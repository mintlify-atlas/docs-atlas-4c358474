## Overview

Bar Galileo uses environment variables for configuration management. Variables are loaded from a `.env` file located at `bar_galileo/bar_galileo/.env` using the `python-dotenv` package.

## Environment File Location

```bash
bar_galileo/
└── bar_galileo/
    ├── .env              # Environment configuration
    ├── settings.py
    ├── wsgi.py
    └── asgi.py
```

## Quick Setup

<Steps>
  <Step title="Create .env file">
    Create the environment file in the correct location:
    ```bash
    touch bar_galileo/bar_galileo/.env
    ```
  </Step>
  
  <Step title="Add required variables">
    Copy the template below and customize values:
    ```bash
    nano bar_galileo/bar_galileo/.env
    ```
  </Step>
  
  <Step title="Restart Django">
    Restart the server to load new configuration:
    ```bash
    python manage.py runserver
    ```
  </Step>
</Steps>

## Environment Variables Reference

### Core Django Settings

<ParamField path="DEBUG" type="boolean" default="True">
  Enable or disable Django debug mode.
  
  **Accepted values**: `1`, `true`, `yes`, `True` (case-insensitive)
  
  **Production**: Must be `False`
</ParamField>

<ParamField path="secret_key" type="string" required>
  Django secret key for cryptographic signing.
  
  **Security**: Keep this value secret. Never commit to version control.
  
  **Generation**: Use Django's `get_random_secret_key()` or:
  ```bash
  python -c 'from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())'
  ```
</ParamField>

<ParamField path="ALLOWED_HOSTS" type="string" default="* (if DEBUG=True)">
  Comma-separated list of host/domain names that Django can serve.
  
  **Format**: `localhost,example.com,www.example.com`
  
  **Example**: `localhost,bar-galileo.com,192.168.1.100`
</ParamField>

<Warning>
  In production, `ALLOWED_HOSTS` must be explicitly set to your domain names. Never use `*` in production.
</Warning>

### Database Configuration

<ParamField path="DB_NAME" type="string" default="bar_galileo">
  MySQL/MariaDB database name.
</ParamField>

<ParamField path="DB_USER" type="string" default="bar_galileo_user">
  MySQL/MariaDB database user.
</ParamField>

<ParamField path="DB_PASSWORD" type="string" default="Galileo2025">
  MySQL/MariaDB database password.
  
  **Security**: Use a strong password in production.
</ParamField>

<ParamField path="DB_HOST" type="string" default="localhost">
  MySQL/MariaDB database host.
  
  **Examples**: `localhost`, `127.0.0.1`, `db.example.com`
</ParamField>

<ParamField path="DB_PORT" type="string" default="3306">
  MySQL/MariaDB database port.
</ParamField>

### Email Configuration

<ParamField path="emailHost" type="string" required>
  Gmail email address for sending system emails.
  
  **Example**: `bargalileo07@gmail.com`
</ParamField>

<ParamField path="emailPassword" type="string" required>
  Gmail App Password (not your regular Gmail password).
  
  **How to generate**: 
  1. Enable 2-factor authentication on your Google account
  2. Visit https://myaccount.google.com/apppasswords
  3. Generate a new app password
  4. Use the 16-character password in this field
</ParamField>

<Note>
  Gmail App Passwords are different from your regular password. You must enable 2FA to generate app passwords.
</Note>

## Environment File Templates

### Development Template

<CodeGroup>
```bash .env (Development)
# Django Core Settings
DEBUG=True
secret_key=your-secret-key-here
ALLOWED_HOSTS=localhost,127.0.0.1

# Database Configuration
DB_NAME=bar_galileo
DB_USER=bar_galileo_user
DB_PASSWORD=your_password_here
DB_HOST=localhost
DB_PORT=3306

# Email Configuration
emailHost=your-email@gmail.com
emailPassword=your-app-password-here
```

```bash .env (Production)
# Django Core Settings
DEBUG=False
secret_key=CHANGE-THIS-TO-A-SECURE-RANDOM-KEY
ALLOWED_HOSTS=bar-galileo.com,www.bar-galileo.com

# Database Configuration
DB_NAME=bar_galileo_prod
DB_USER=bar_galileo_user
DB_PASSWORD=STRONG-SECURE-PASSWORD-HERE
DB_HOST=localhost
DB_PORT=3306

# Email Configuration
emailHost=bargalileo07@gmail.com
emailPassword=APP-PASSWORD-16-CHARS
```
</CodeGroup>

## Production Security Checklist

<Warning>
  Before deploying to production, verify all security settings:
</Warning>

<Steps>
  <Step title="Set DEBUG=False">
    ```bash
    DEBUG=False
    ```
    This disables debug pages and protects sensitive information.
  </Step>
  
  <Step title="Generate secure SECRET_KEY">
    ```bash
    python -c 'from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())'
    ```
    Use a unique, random key for production.
  </Step>
  
  <Step title="Configure ALLOWED_HOSTS">
    ```bash
    ALLOWED_HOSTS=yourdomain.com,www.yourdomain.com
    ```
    Only include your actual domain names.
  </Step>
  
  <Step title="Use strong database password">
    ```bash
    DB_PASSWORD=STRONG-RANDOM-PASSWORD-HERE
    ```
    Never use the default password in production.
  </Step>
  
  <Step title="Set up SSL/HTTPS">
    Ensure your server has a valid SSL certificate. Security settings are auto-enabled when `DEBUG=False`:
    - `SECURE_SSL_REDIRECT = True`
    - `SESSION_COOKIE_SECURE = True`
    - `CSRF_COOKIE_SECURE = True`
    - `SECURE_HSTS_SECONDS = 31536000`
  </Step>
</Steps>

## Auto-Enabled Production Settings

When `DEBUG=False`, these security features are automatically enabled:

```python settings.py
if not DEBUG:
    SECURE_SSL_REDIRECT = True                    # Force HTTPS
    SESSION_COOKIE_SECURE = True                  # Secure session cookies
    CSRF_COOKIE_SECURE = True                     # Secure CSRF cookies
    SECURE_HSTS_SECONDS = 31536000                # HSTS for 1 year
    SECURE_HSTS_INCLUDE_SUBDOMAINS = True         # HSTS for subdomains
    SECURE_HSTS_PRELOAD = True                    # HSTS preload list
    SECURE_REFERRER_POLICY = 'strict-origin-when-cross-origin'
```

## Environment Variable Loading

Bar Galileo loads environment variables using `python-dotenv`:

```python settings.py
from dotenv import load_dotenv
from pathlib import Path

load_dotenv(dotenv_path=Path(__file__).resolve().parent / '.env')
```

### Variable Access in Code

```python
import os

# Get variable with default
db_name = os.getenv('DB_NAME', 'bar_galileo')

# Get required variable
secret = os.getenv('secret_key')

# Boolean conversion
debug = str(os.getenv('DEBUG', 'True')).lower() in ('1', 'true', 'yes')

# List conversion (comma-separated)
raw_hosts = os.getenv('ALLOWED_HOSTS', '')
hosts = [h.strip() for h in raw_hosts.split(',') if h.strip()]
```

## Common Issues

### .env File Not Found

If environment variables aren't loading:

<Steps>
  <Step title="Check file location">
    The `.env` file must be in `bar_galileo/bar_galileo/.env` (same directory as `settings.py`)
  </Step>
  
  <Step title="Verify file name">
    Ensure the file is named exactly `.env` (not `env.txt` or `.env.example`)
  </Step>
  
  <Step title="Check file permissions">
    ```bash
    chmod 600 bar_galileo/bar_galileo/.env
    ```
  </Step>
</Steps>

### Database Connection Errors

<AccordionGroup>
  <Accordion title="Access denied for user">
    Check `DB_USER` and `DB_PASSWORD` are correct:
    ```bash
    mysql -u bar_galileo_user -p
    ```
    Enter the password from your `.env` file.
  </Accordion>
  
  <Accordion title="Unknown database">
    Create the database:
    ```sql
    CREATE DATABASE bar_galileo CHARACTER SET utf8mb4;
    ```
  </Accordion>
  
  <Accordion title="Can't connect to MySQL server">
    Check `DB_HOST` and `DB_PORT`:
    ```bash
    mysql -h localhost -P 3306 -u bar_galileo_user -p
    ```
  </Accordion>
</AccordionGroup>

### Email Configuration Issues

<AccordionGroup>
  <Accordion title="Authentication failed">
    - Ensure you're using a Gmail App Password, not your regular password
    - Enable 2-factor authentication on your Google account
    - Generate a new app password at https://myaccount.google.com/apppasswords
  </Accordion>
  
  <Accordion title="SMTPException: STARTTLS extension not supported">
    Check that `EMAIL_PORT=587` and `EMAIL_USE_TLS=True` are set correctly in settings.py
  </Accordion>
</AccordionGroup>
